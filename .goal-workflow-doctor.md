## What to build
A new capability **`workflow-doctor`** that diagnoses a failing Smithers workflow from its own past-run logs and proposes/applies a minimal fix to the workflow source file. This is a "workflow for fixing workflows."

### Behavior (pipeline)
1. **Gather evidence** — given an input `targetWorkflow` (a capability/workflow slug, e.g. `smart-contract-audit`), fetch recent runs of that workflow from the Hub API and keep the FAILED/errored ones, with their error messages, normalized error fingerprints, failed node ids, and log tails. REUSE the exact Hub-fetch + secret-redaction approach already in `workflow-templates/workflows/run-knowledge-builder.tsx` (same HUB_URL/HUB_TOKEN env, same REDACTION_RULES). Do not reinvent redaction.
2. **Read the workflow source** — read `workflow-templates/workflows/<targetWorkflow>.tsx` and include it (redacted) as context.
3. **Diagnose** — a ClaudeCodeAgent reads the redacted errors + source and returns a structured diagnosis: rootCause, failingNode, the smallest correct fix, and a confidence. Distinguish deterministic code bugs (null/undefined reads, bad schema, typos) from transient/infra failures (it should say "no code fix" when the failures are transient).
4. **Fix (gated)** — if `apply` is true AND the diagnosis is a deterministic code bug with sufficient confidence, an implementation agent edits ONLY `workflow-templates/workflows/<targetWorkflow>.tsx` with the smallest correct change (model this on `workflow-templates/workflows/implement-change-gated.tsx`'s contract). Then validate:
   - the edited workflow graphs cleanly: `npx smithers graph workflow-templates/workflows/<targetWorkflow>.tsx` (or the repo's equivalent graph command — discover the correct invocation) must exit 0.
   - `pnpm test` passes (if a test script exists).
   If `apply` is false (default), STOP after diagnosis and return the proposed fix + a unified diff preview WITHOUT writing the file.
5. **Approval gate** — the capability's `approvalPolicy` must be `{ required: true, notifyTelegram: true, reason: "..." }` so a human approves before any fix is committed. Do not auto-commit/push without approval.

### Input schema (zod)
- `targetWorkflow: string` (required) — workflow/capability slug to diagnose
- `lookbackHours: number` default 168
- `count: number` default 20 (max 50)
- `apply: boolean` default false — false = diagnose only; true = apply the minimal fix behind the approval gate
- `focus: string` optional

### Output schema
- `diagnosis` (object: rootCause, failingNode, fixSummary, confidence, isDeterministic)
- `evidence` (object: runsSampled, failedCount, topFingerprints)
- `proposedDiff` (string)
- `applied` (boolean)
- `graphOk` (boolean)
- `testResult` (object)
